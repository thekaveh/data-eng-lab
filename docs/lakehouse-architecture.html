Lakehouse Architecture

Apache Iceberg lakehouse on MinIO · Iceberg REST catalog · Spark + Trino + Redpanda

Compute Engine

  • • Spark 4.1.2 (Scala 2.13, Iceberg 1.11.0)
  • • Spark Connect + Standalone Master + 2x Workers
  • • Trino 482 for SQL queries over Iceberg
  • • Redpanda v26 for streaming sinks
  • • Airflow 3.2.2 orchestrates Spark jobs

Storage & Catalog

  • • MinIO (S3-compatible object storage)
  • • Iceberg REST catalog (Postgres backend)
  • • Catalog: lakehouse · Namespaces: bronze/silver/gold
  • • 4 buckets: landing, lakehouse, jars, checkpoints

Medallion Pattern

  • • Bronze: raw landing, minimal cleaning
  • • Silver: dedup, validate, schema enforced
  • • Gold: business marts, BI-ready aggregations
  • • ACID transactions, time travel, MERGE INTO
