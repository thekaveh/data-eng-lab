NYC Taxi ETL — Spark Application Architecture

Raw Parquet landing → Jenkins CI → Airflow CD → Iceberg bronze layer

CI Pipeline (Jenkins)

  • SCM poll / webhook on main branch
  • mvn test (ScalaTest unit tests)
  • mvn package (shaded JAR via maven-shade-plugin)
  • JAR published to MinIO jars bucket

CD Pipeline (Airflow)

  • SparkSubmitOperator, cluster deploy mode
  • JAR pulled from s3a://jars/nyc-taxi-etl/
  • spark.sql.extensions set to Iceberg extensions
  • S3A credentials on Spark cluster for catalog

Transform: TaxiTransforms

  • Drop null pickup rows
  • Drop non-positive passenger counts
  • Add trip_date = to_date(created_at)
  • Output: lakehouse.bronze.nyc_taxi_trips
